PROJECT OVERVIEW:
-----------------
This is a Java Spring Boot web application for intelligent university timetable
scheduling and classroom allocation. The system uses three advanced algorithms
to optimize scheduling, detect conflicts, and provide auto-completion features.

TECHNOLOGY STACK:
-----------------
- Backend: Java 21, Spring Boot 3.1.5
- Build Tool: Maven 3.9+
- Frontend: HTML5, CSS3, JavaScript (Vanilla)
- Architecture: RESTful API, MVC Pattern
- Server Port: 8082

ALGORITHMS IMPLEMENTED:
-----------------------
1. AVL TREE (AVLTree.java)
   - Purpose: Time-based scheduling with conflict detection
   - Features:
     * Maintains sorted schedule by time
     * O(log n) insertion, search, and deletion
     * Automatic balancing for optimal performance
     * Conflict detection for professors and rooms
   
2. TRIE (Trie.java)
   - Purpose: Auto-completion for courses and rooms
   - Features:
     * Fast prefix-based search
     * O(m) search time where m = prefix length
     * Real-time suggestions as user types
     * Supports course codes and room numbers
   
3. GREEDY ALGORITHM (GreedyRoomAllocator.java)
   - Purpose: Optimal classroom allocation
   - Features:
     * Selects smallest suitable room (capacity-based)
     * O(n log n) complexity for room sorting
     * Maximizes room utilization
     * Prevents over-allocation

API ENDPOINTS:
--------------
GET  /api/courses                    - Get all courses
GET  /api/professors                 - Get all professors
GET  /api/rooms                      - Get all rooms
GET  /api/timeslots                  - Get all time slots
GET  /api/schedule/all               - Get all scheduled entries (sorted by AVL Tree)
GET  /api/schedule/day?day=Monday    - Get schedule for specific day
GET  /api/autocomplete/course?prefix=CS - Course auto-complete (Trie)
GET  /api/autocomplete/room?prefix=101 - Room auto-complete (Trie)
GET  /api/rooms/available?timeSlotId=T1 - Get available rooms for time slot (Greedy)
POST /api/schedule                   - Schedule a new class (uses all 3 algorithms)
     Request Body: {
       "courseId": "C1",
       "professorId": "P1",
       "timeSlotId": "T1"
     }

HOW TO RUN:
-----------

Prerequisites:
  - Java JDK 21 installed
  - Maven 3.9+ installed (or use Maven Wrapper)
  - JAVA_HOME and MAVEN_HOME environment variables set

Option 1: Using Maven (if installed)

Option 3: Build and Run JAR

  java -jar target/timetable-system-1.0.0.jar

Access Application:
  Open browser: http://localhost:8082

ALGORITHM COMPLEXITY ANALYSIS:
-------------------------------
AVL Tree:
  - Insert: O(log n)
  - Search: O(log n)
  - Delete: O(log n)
  - Conflict Check: O(log n)
  - Space: O(n)

Trie:
  - Insert: O(m) where m = word length
  - Search: O(m)
  - Auto-complete: O(m + k) where k = number of results
  - Space: O(ALPHABET_SIZE × N × M) where N = number of words, M = avg length

Greedy Room Allocator:
  - Room Sorting: O(n log n) where n = number of rooms
  - Availability Check: O(m) where m = scheduled entries
  - Total: O(n log n + m)
  - Space: O(n)

TROUBLESHOOTING:
----------------
Port Already in Use:
  Edit src/main/resources/application.properties
  Change: server.port=8082 to server.port=8083

Build Fails:
  - Verify Java 21: java -version
  - Verify Maven: mvn -version
  - Check JAVA_HOME and MAVEN_HOME environment variables

Frontend Not Loading:
  - Hard refresh browser: Ctrl+F5 (Windows) or Cmd+Shift+R (Mac)
  - Clear browser cache
  - Restart Spring Boot server

Maven Not Found:
  - Run install-maven.ps1 script (as Administrator on Windows)
  - Or manually install Maven and set MAVEN_HOME
